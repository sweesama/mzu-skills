# OpenClaw 水产市场操作指南

这是一个围绕 `openclawmp` CLI 的市场操作型 skill，核心作用是把水产市场相关任务整理成清晰、可执行的处理流程。它覆盖从环境检查到资产搜索、安装、发布前校验的一整套常见场景，帮助用户在面对市场资产时更快判断当前该查什么、缺什么、下一步该做什么。

## 这个资产能做什么

- 检查 `openclawmp` CLI 是否存在，以及当前机器是否具备基础使用条件
- 区分哪些操作依赖本地凭证，帮助用户判断 token 或登录状态是否满足安装、发布等任务
- 处理市场资产的搜索、详情查看、安装、卸载等常见操作，并以真实 CLI 输出为准
- 在发布本地资产前检查目录结构和关键文件，尤其是 `.metadata.json` 与 `README.md` 是否满足最小要求
- 当用户不确定资产类型时，快速判断更适合发布为 `skill`、`plugin`、`trigger`、`channel` 或 `experience`
- 在遇到 CLI 缺失、凭证无效、目录错误、元数据缺失等问题时，明确归类失败原因并给出下一步建议

## 功能特点

- 以实际市场操作任务为中心，而不是只解释命令含义
- 优先遵循本地 `openclawmp` CLI 的真实帮助信息、输出和报错
- 只收集完成当前任务真正需要的输入，减少无关确认
- 在发布相关任务里先做本地校验，再决定是否继续执行
- 在补全发布文件时，优先沿用作者已有材料中的惯用语言

## 适用场景

- 需要快速确认当前机器能否执行 `openclawmp` 相关任务
- 需要判断某个市场操作是否依赖本地鉴权信息
- 需要搜索、查看、安装或卸载水产市场中的资产
- 需要在发布本地资产前检查目录、元数据和说明文件是否合规
- 需要为待发布资产判断合适的 `assetType`
- 需要在市场操作失败时快速定位是 CLI、凭证、目录还是元数据问题

## 配置说明

使用该 skill 前，应确保本机已安装 `openclawmp` CLI。

如果当前操作依赖鉴权，请准备可用的本地 token。默认凭证路径如下：

- macOS / Linux：`~/.openclaw/hub-credentials.json`
- Windows：`%USERPROFILE%\\.openclaw\\hub-credentials.json`

也可以在执行命令时通过 `--token-file <path>` 指定其他凭证文件。
